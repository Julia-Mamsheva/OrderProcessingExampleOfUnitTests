<h1 align = "center"> OrderProcessing Library </h1>
<h2>Описание</h2>
Библиотека OrderProcessing предназначена для обработки заказов и управления запасами продуктов. Она предоставляет функциональность для обработки заказов, расчета общей стоимости и получения текущего состояния запасов. Библиотека реализована в виде .NET DLL и может быть использована в различных приложениях, требующих управления запасами и обработки заказов

<h2>Класс OrderProcessor</h2>
<h3>Конструктор</h3>

 *    `OrderProcessor(Dictionary<string, int> inventory)`: Инициализирует новый экземпляр класса OrderProcessor с заданным словарем запасов, где ключом является название продукта, а значением — количество на складе

   <h3>Методы</h3>
 
 1.   `ProcessOrder(string product, int quantity)`
   
   *    Обрабатывает заказ на указанный продукт и количество
   *    **Параметры**:

        *    `product`: Название продукта (строка)
        *    `quantity`: Количество продукта (целое число)
   *   **Возвращает**: Объект `OrderResult`, содержащий информацию о результате обработки заказа
   *   **Исключения**:
        *   `ArgumentException`: Выбрасывается, если название продукта пустое или null, или если количество меньше или равно нулю
   *   **Логика**:
        *   Проверяет наличие продукта в запасах и достаточность количества для выполнения заказа. Уменьшает количество на складе, если заказ успешно обработан
 2.  `lculateTotalPrice(string product, int quantity, decimal price)`

  *    Рассчитывает общую стоимость заказа.
  *    **Параметры**:
       *    `product`: Название продукта (строка)
       *    `quantity`: Количество продукта (целое число)
       *    `price`: Цена за единицу продукта (десятичное число)
  *    **Возвращает**: Общую стоимость заказа (десятичное число)
  *    **Исключения**:
       *    `ArgumentException`: Выбрасывается, если цена меньше или равна нулю
  *    `GetInventory()`
       *    Получает текущее состояние запасов
       *    **Возвращает**: Копию словаря запасов, чтобы избежать изменения оригинального словаря
<h2>Использование</h2>
Для использования библиотеки `OrderProcessing` необходимо добавить ссылку на DLL в проект. После этого можно создать экземпляр класса OrderProcessor, передав ему словарь запасов, и использовать предоставленные методы для обработки заказов и управления запасами

<h2>Тестирование</h2>
Библиотека `OrderProcessing` включает в себя набор автоматизированных тестов, реализованных с использованием фреймворка NUnit. Тесты находятся в классе `OrderProcessorTests` и охватывают следующие аспекты:

  *    Проверка успешной обработки корректных заказов
  *    Обработка ошибок при попытке заказа несуществующих продуктов или недостаточного количества на складе
  *    Проверка корректности обработки исключений для недопустимых входных данных (например, нулевое или отрицательное количество, пустое или null название продукта)
  *    Проверка правильности расчета общей стоимости заказа
  *    Убедиться, что метод `GetInventory` возвращает копию запасов и не изменяет оригинальный словарь
  *    Проверка, что запасы корректно обновляются после успешных заказов и не могут стать отрицательными
  *    Проверка чувствительности поиска продукта к регистру
